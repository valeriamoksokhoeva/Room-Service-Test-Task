# Room Booking Service

![CI](https://github.com/valeriamoksokhoeva/Room-Service-Test-Task/actions/workflows/ci.yml/badge.svg)

Сервис бронирования переговорок.
Администраторы создают переговорки и настраивают расписание их доступности.
Система автоматически формирует слоты для бронирования.
Сотрудники просматривают свободные слоты и создают или отменяют брони.

## Возможности

**Администратор:**
- Создание переговорок
- Настройка расписания доступности (дни недели, время начала и окончания)
- Просмотр всех броней с пагинацией

**Сотрудник:**
- Просмотр списка переговорок
- Просмотр доступных слотов по переговорке и дате
- Бронирование слота
- Отмена своей брони
- Просмотр своих броней

## Решения

**Генерация слотов**
При создании расписания слоты генерируются на 30 дней вперёд и сохраняются в БД.
Крон-джоб каждые 24 часа добавляет слоты на следующий день.
Такой подход выбран потому что самый нагруженный эндпоинт `/rooms/{roomId}/slots/list`
просто читает готовые данные из БД без вычислений на лету.
Индекс `slots(room_id, start_at)` обеспечивает быстрый поиск.

**Бронирование**
При создании брони используется транзакция с `SELECT FOR UPDATE` для блокировки слота.
Частичный уникальный индекс `bookings(slot_id) WHERE status = 'active'`
гарантирует что один слот не может быть забронирован дважды даже при конкурентных запросах.

**Отмена брони**
Операция идемпотентна — повторный вызов на уже отменённой брони
возвращает 200 с актуальным состоянием.

**Conference Service**
При передаче `createConferenceLink: true` выполняется запрос к внешнему сервису.
При сбое внешнего сервиса бронь создаётся без ссылки — ошибка логируется.
Бронирование важнее ссылки на конференцию.
